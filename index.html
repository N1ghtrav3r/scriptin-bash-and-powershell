Shebang

Vertelt het systeem welke interpreter het script uitvoert. Moet op de allereerste regel staan.

# Dit is commentaar

chmod +x script.sh        # uitvoerrechten geven

bash script.sh            # ook geldig, zonder chmod
bash -x script.sh         # debug modus

Stringmanipulatie

tekst="Hallo Wereld"

echo ${tekst,,}          # kleine letters
echo ${tekst^^}          # hoofdletters
echo ${tekst:0:5}        # substring: "Hallo"
echo ${tekst/Hallo/Dag}  # vervangen

grep & regex

grep -E "[0-9]+" bestand.txt    # extended regex
grep -v "patroon" bestand.txt    # NIET matchende regels
grep -c "patroon" bestand.txt    # tel matches
grep -n "patroon" bestand.txt    # toon regelnummers

sed & awk

awk '{print $1}' bestand.txt          # eerste kolom
awk -F: '{print $1}' /etc/passwd      # scheidingsteken ':'

Exitcodes gebruiken

0 = succes, alles anders = fout.

ls /bestaat-niet
echo $?           # exitcode vorige commando

if ping -c 1 8.8.8.8 >/dev/null 2>&1; then
    echo "Netwerk OK"
else
    echo "Geen verbinding"
fi

exit 0    # succes
exit 1    # fout
OperatorUitleg
cmd1 && cmd2cmd2 alleen als cmd1 slaagde
cmd1 || cmd2cmd2 alleen als cmd1 faalde
$?Exitcode van het laatste commando

Veelgemaakte fouten

Geen spaties rond =: naam="x" is correct, naam = "x" geeft een fout
Vergeet "$var" niet te quoten — spaties in waarden breken commando's

try / catch / finally

try {
    Get-Content "bestaat-niet.txt" -ErrorAction Stop
    Write-Host "Bestand gelezen"
}
catch {
    Write-Host "Fout: $($_.Exception.Message)"
}
finally {
    Write-Host "Wordt altijd uitgevoerd"
}
ParameterUitleg
-ErrorAction StopFout wordt exception (nodig voor try/catch)
-ErrorAction SilentlyContinueFout negeren
$Error[0]Laatste fout opvragen
$?True/False: was laatste commando OK?

Veelgemaakte fouten

-ErrorAction Stop is vereist anders vangt try/catch niets op!
Execution policy blokkeert scripts? Set-ExecutionPolicy RemoteSigned als admin
Write-Host vs Write-Output: Host toont op scherm maar gaat niet in de pipeline
